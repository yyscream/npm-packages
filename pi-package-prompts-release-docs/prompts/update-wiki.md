description: Apply branch changes to wiki docs comprehensively with minimal structural churn.
---
Update wiki content based on current branch changes.

Process:
1. Identify documentation-relevant changes in code/config/behavior.
2. Map each change to existing wiki sections.
3. Update in-place where possible; create new sections only when truly needed.

Quality rules:
- complete but concise
- user-friendly wording
- no duplicated content across sections
- include usage/impact notes when behavior changed

Output:
- Updated wiki markdown files
- Short summary of pages/sections touched
