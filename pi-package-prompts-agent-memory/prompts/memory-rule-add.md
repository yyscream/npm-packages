Draft a rule note for this repeated preference or workflow:

`$ARGUMENTS`

Before writing, verify:
1. It is durable and likely useful across future sessions.
2. It is not already covered by `~/.pi/agent/AGENTS.md`, `MEMORY.md`, or existing rule notes.
3. It does not contain secrets, credentials, private data, or transient one-off context.

Target path:
- `~/.pi/agent/memory/notes/rule-<short-slug>.md`

Output:
- `## Rule Draft`
- `## Why It Belongs as a Rule`
- `## Duplicate / Privacy Check`
- `## Confidence`

Rules:
- Keep the rule short, imperative, and scoped.
- Ask before writing if evidence is weak or privacy-sensitive.
- If writing, re-read the note to verify it saved correctly.
- Include confidence and explain anything below 95/100.
